Connect SMARTCoach to GHL SaaS plans.

This page is written for the sales/onboarding platform. It explains the plan connection without exposing private tokens or coach codes.

Customer chooses a plan

Essential, Pro 25, Pro 100, Pro 200, or custom Pro Unlimited maps to the SMARTCoach account record and subscription access state.

GHL provisions the account

GHL SaaS creates the sub-account and applies the SMARTCoach snapshot for that customer.

Account key connects the app

The snapshot should include a custom value named account_key and a SMART Trak custom menu link using that value.

Automation saves billing state

The GHL workflow webhook sends plan, assistant coach seats, subscription status, billing cadence, source, and account identifiers.

Support completes setup

After the sub-account exists, support enters the Location ID, Private Integration Token, and shared coach code setup in SMARTCoach onboarding.

Coach invite goes out

After validation, the head coach receives the SMARTCoach link, account key, and shared coach code for active staff.

Custom menu link for SMART Trak.

The SMART Trak custom menu link should point coaches into the right account-scoped dashboard. In the GHL snapshot, use the account key custom value.

Menu URL:
https://app.smartcoach-pro.com/dashboard.html?account={{custom_values.account_key}}&embed=1

Recurring subscription payload should include only safe billing/account fields.

  • eventType
  • accountKey
  • locationId
  • plan
  • coachSeats
  • subscriptionStatus
  • billingCadence
  • source
  • automationSecret or secret query parameter

Do not send private setup values in billing automation.

Recurring subscription payloads should not include private integration tokens or coach access codes. Billing updates should not reset coach codes or count against code reset limits.

  • Private Integration Token is entered during support setup
  • The shared coach code is created intentionally during onboarding
  • Coach code changes are limited to 2 times per month per account
  • A saved code change invalidates old signed Pro sessions
  • GHL controls the 7-day trial and sends trialing, active, or blocked billing status
